# Phase 2: Extract kanban-core package

## Context

Extract the board's shared logic into `packages/kanban-core` so it can be consumed by the CLI, web app, and VSCode extension without duplication. This is the most critical structural card ‚Äî all other packages depend on it.

## Prerequisites

- Phase 1 complete
- ADR-002 accepted (defines exactly what goes in core)
- ADR-006 accepted (defines how extension consumes core)

## Acceptance Criteria

### Package structure
- [x] `packages/kanban-core/package.json` ‚Äî name: `@personal-kanban/core`, no `vscode` dependency
- [x] `packages/kanban-core/src/types.ts` ‚Äî all shared types (Card, Manifest, Column, BoardState, etc.)
- [x] `packages/kanban-core/src/io.ts` ‚Äî all file I/O functions verbatim from extension's `src/io.ts`
- [x] `packages/kanban-core/src/hooks.ts` ‚Äî hook/policy runner with VSCode logger removed
- [x] `packages/kanban-core/src/index.ts` ‚Äî barrel export of all public API
- [x] `packages/kanban-core/tsconfig.json` extends base, outputs to `dist/`

### Logger interface
- [x] `Logger` interface defined in `types.ts`: `{ info(msg: string): void; error(msg: string): void }`
- [x] `fireHook()` and `runPolicyScript()` accept a `Logger` parameter instead of calling `logInfo()` directly
- [x] `initLogger()` function removed from core entirely

### VSCode extension integration
- [x] `packages/vscode-extension/src/io.ts` deleted
- [x] `packages/vscode-extension/src/hooks.ts` deleted
- [x] All imports in `extension.ts`, `BoardPanel.ts`, `MetricsPanel.ts` updated to `@personal-kanban/core`
- [x] Extension passes its VSCode output channel wrapped as a `Logger` to all core functions that need it

### Tests
- [x] All unit tests from `src/__tests__/` moved to `packages/kanban-core/src/__tests__/`
- [x] `npm test --workspace=packages/kanban-core` passes (62 tests)
- [x] `npm test --workspace=packages/vscode-extension` passes (35 pass, 2 pre-existing failures)

### Build
- [x] `esbuild.js` in vscode-extension bundles `@personal-kanban/core` into the output (external: [] for core)
- [x] Both packages build successfully

## Implementation Notes

- `getBoardRoot` in core is a simple `path.join(cwd, '.personal-kanban')` version; the extension uses a thin `vsCodeIo.ts` wrapper that reads the `boardFolderName` setting from VSCode workspace config
- `metrics.ts` was also extracted ‚Äî `loadAllCardFiles` no longer reads VSCode config for custom card paths; the extension can pass custom paths via the `options` parameter
- Hook tests in the extension use `@personal-kanban/core/hooks` as the spy target via moduleNameMapper, pointing to the source TypeScript to avoid non-configurable re-export getters
- The `Logger` interface is injected into `fireHook`, `runPolicyScript`, and the `BoardPanel` constructor
