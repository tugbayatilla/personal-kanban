#ui #theming

# Card theming system

Add a `theme` field to `manifest.json` that controls board card appearance. Three built-in themes: `default`, `compact`, `minimal`. Also support a custom CSS file path.

## Acceptance Criteria

- [ ] `Manifest` type has `theme?: string`
- [ ] Three built-in theme CSS files: `default.css`, `compact.css`, `minimal.css` in `media/themes/`
- [ ] `BoardPanel.ts` loads all theme stylesheets in webview HTML
- [ ] `board.js` applies `theme-<name>` class to `document.body` on setState
- [ ] Custom `.css` path injects CSS via `injectTheme` message
- [ ] Tests pass for theme field round-trip in `readManifest`

## Implementation Notes

- `theme?: string` was already present in `packages/kanban-core/src/types.ts` — no change needed
- Created `packages/vscode-extension/media/themes/default.css`, `compact.css`, `minimal.css` with scoped CSS class selectors (`body.theme-compact`, `body.theme-minimal`)
- Updated `BoardPanel.ts` (`packages/vscode-extension/src/BoardPanel.ts`):
  - Added `import * as fs from 'fs'`
  - `_sendState()` sends `injectTheme` message with file contents when `manifest.theme` ends with `.css`
  - `_getHtml()` includes all three theme `<link>` tags and uses `'unsafe-inline'` in the `style-src` CSP directive to allow the injected `<style>` element
- Updated `packages/vscode-extension/media/board.js`:
  - Added `applyTheme(theme)` function that removes existing `theme-*` classes and adds the new one
  - `setState` handler calls `applyTheme(msg.manifest?.theme)` after render
  - New `injectTheme` message case injects CSS into a `<style id="custom-theme">` element
- Added three new tests to `packages/kanban-core/src/__tests__/io.test.ts` covering `theme` round-trip (`compact`, undefined, and `.css` path)
- All 65 kanban-core tests pass; 2 pre-existing `boardPanel` failures remain (unrelated to this change)
