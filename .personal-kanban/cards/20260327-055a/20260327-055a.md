# card info in manifest

Implemented 2026-03-28: moved card IDs into each column object (`column.cards[]`), removing the top-level `cards` map. Manifest version bumped to 2.

the card ids for each column can be kept inside of the columns section.
example
```
{
      "id": "backlog",
      "label": "Backlog",
      "wip_limit": null,
      "cards": [<card_id_1>,<card_id_2>]

    }
```
