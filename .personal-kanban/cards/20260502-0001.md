# Phase 1: Monorepo foundation setup

## Context

This is the first implementation card after ADRs are written. It converts the existing single-package repository into an npm workspaces monorepo and moves the VSCode extension into `packages/vscode-extension`. No logic changes — structure only.

## Prerequisites

- ADR-001 accepted
- ADR-002 accepted

## Acceptance Criteria

- [ ] Root `package.json` has `"workspaces": ["packages/*"]` and no `main`/`activationEvents` fields (those belong in the extension package)
- [ ] `packages/vscode-extension/` contains all current source: `src/`, `media/`, `esbuild.js`, `tsconfig.json`, `.vscodeignore`
- [ ] `packages/vscode-extension/package.json` is the current root `package.json` content, adjusted for its new path
- [ ] `tsconfig.base.json` at root defines shared compiler options: `strict: true`, `target: ES2020`, `module: CommonJS`, `esModuleInterop: true`, `skipLibCheck: true`
- [ ] Each package `tsconfig.json` extends `../../tsconfig.base.json`
- [ ] `npm install` from root resolves without errors
- [ ] `npm run build --workspace=packages/vscode-extension` produces `dist/extension.js` identical to the current build
- [ ] `npm test --workspace=packages/vscode-extension` passes all existing tests
- [ ] Git history preserved — use `git mv` not file copy, so `git log --follow` works on moved files

## Out of Scope

- Do not extract core yet (Phase 2)
- Do not add CLI or web packages yet
- Do not change any TypeScript source logic
