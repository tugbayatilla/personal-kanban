# Phase 3: kanban-cli package

## Context

Build the command-line interface for personal-kanban. The CLI is a standalone npm package that reads and writes the `.personal-kanban/` directory in the current working directory. It is the primary non-VSCode interface to the board.

## Prerequisites

- Phase 2 complete (kanban-core available)
- ADR-003 accepted (CLI framework chosen)

## Commands to implement

| Command | Description |
|---------|-------------|
| `kanban init` | Bootstrap `.personal-kanban/` in cwd (calls core's init logic) |
| `kanban status` | Print board summary: columns with card counts and WIP status |
| `kanban add <title> [--column <id>] [--tag <tag>]` | Create a new card |
| `kanban move <card-id> <column-id>` | Move a card to a column |
| `kanban done <card-id>` | Move a card to Done column |
| `kanban archive` | Archive all cards in Done column |
| `kanban serve [--port <n>]` | Start the web app (Phase 4) |

## Acceptance Criteria

### Package structure
- [x] `packages/kanban-cli/package.json` — name: `@personal-kanban/cli`, `bin: { "kanban": "./dist/index.js" }`
- [x] `packages/kanban-cli/src/index.ts` — CLI entrypoint with shebang `#!/usr/bin/env node`
- [x] All commands import from `@personal-kanban/core` only — no direct file I/O in CLI layer

### Behavior
- [x] `kanban status` output example:
  ```
  Backlog      [5 cards]
  Refined      [2 cards]
  Planning     [1 card]
  In Progress  [1/2] ⚠ near limit
  Review       [0 cards]
  Done         [3 cards]
  ```
- [x] All commands accept `--json` flag and output machine-readable JSON (same data, different format)
- [x] Error messages go to stderr; success output to stdout
- [x] Exit code 0 on success, non-zero on failure
- [x] Commands fire the same hooks as the VSCode extension (uses `fireHook()` from core)

### Distribution
- [ ] `npm pack` produces a tarball installable via `npm install -g`
- [ ] `npx @personal-kanban/cli status` works without global install

### Tests
- [x] Each command has a unit test using a temp directory as the board root
- [x] Tests do not spawn subprocesses — call command handlers directly
- [x] `npm test --workspace=packages/kanban-cli` passes

## Implementation Notes

- Package created at `packages/kanban-cli/` with `commander` as the CLI framework
- All commands are synchronous wrappers around kanban-core's sync API (`loadBoardState`, `readCard`, `writeCard`, etc.)
- `fireHook` requires the manifest as a second argument (signature: `fireHook(boardRoot, manifest, event, payload, logger)`)
- `loadBoardState` is synchronous and returns cards keyed by ID; column card lists are in `col.cards`
- Card type uses `metadata` object: `card.metadata.column`, `card.metadata.done_at`, etc.
- 21 tests across 5 test files — all passing
