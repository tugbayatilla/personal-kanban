# Atomic manifest writes with exclusive file locking

Implemented 2026-03-31. withLock() uses O_EXCL flag for cross-process exclusion; all card+manifest operations now held under a shared lock.

Manifest writes must be atomic with card file operations (create/delete/archive).
Replace the advisory PID-based lock with an exclusive O_EXCL lock that blocks
concurrent writers. Expose a `withLock` wrapper so callers can lock across both
the card file operation and the manifest write as a single atomic unit.
