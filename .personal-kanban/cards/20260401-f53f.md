# default scripts not wired up in ensureWorkspaceSettings

`ensureWorkspaceSettings` (called by `Init Board`) writes `scripts: {}` and `hooks: {}`.
The built-in scripts in `.personal-kanban/scripts/` are never connected to the extension's hook system unless the user wires them manually.

Default wiring should be:
- `card.reviewed` → `card-reviewed.js` (notification when card moves to Review)
- `wip.violated` → `wip-alert.js` (notification when WIP limit exceeded)

## Plan

Wire default scripts/hooks in `initBoard` in `extension.ts` after writing the manifest, only when the workspace setting is not already set.

## Summary

Added default scripts/hooks wiring to `initBoard` in [extension.ts](src/extension.ts). After creating the manifest, it now writes `card-reviewed` and `wip-alert` script definitions and their hook bindings to workspace settings — but only when those keys are not already configured, so existing setups are unaffected.
