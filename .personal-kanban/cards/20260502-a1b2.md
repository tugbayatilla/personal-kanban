# Add Planning column between Refined and In Progress

## Context

The board currently has: Backlog → Refined → In Progress → Review → Done.
A Planning column is needed between Refined and In Progress to represent active design/spec work before implementation begins. Planning is an active column (work is happening, not queued).

## Acceptance Criteria

- [ ] `manifest.json` has a new column `{ "id": "planning", "label": "Planning", "index": 2, "wip_limit": null, "policies": [] }` inserted between `refined` (index 1) and `in-progress` (now index 3)
- [ ] All subsequent column indexes are incremented by 1 (in-progress → 3, review → 4, done → 5)
- [ ] `column_stamps.active_at` remains pointing to `in-progress` (Planning does NOT set active_at — it is a thinking column, not a doing column)
- [ ] Board UI renders the new column in correct position
- [ ] Cards can be moved into and out of Planning without policy errors
- [ ] `no-pullback` policy correctly treats Planning as forward from Refined and backward from In Progress

## Notes

- Do NOT add an `entry:planning` policy script — this column has no gate
- The wip_limit for Planning is intentionally null (unconstrained)
