# Phase 4: kanban-web package

## Context

Build a local web app that serves the kanban board in a browser. The web app reuses the existing board UI from the VSCode extension's `media/` assets, adapted to communicate via HTTP instead of VSCode's postMessage API. Launched via `kanban serve`.

## Prerequisites

- Phase 2 complete (kanban-core available)
- Phase 3 complete (`kanban serve` command stub exists)
- ADR-004 accepted (web server framework chosen)
- ADR-005 accepted (frontend strategy chosen)

## Acceptance Criteria

### Package structure
- [ ] `packages/kanban-web/package.json` — name: `@personal-kanban/web`
- [ ] `packages/kanban-web/src/server.ts` — Express/Fastify/Hono server setup
- [ ] `packages/kanban-web/src/routes/` — one file per route group (board, cards, archive)
- [ ] `packages/kanban-web/src/public/` — static assets (board.js adapted, board.css, adapter.js if applicable)

### REST API
- [ ] `GET  /api/board` → returns `{ manifest, cards }` (same shape as BoardPanel's setState message)
- [ ] `POST /api/cards` → `{ columnId, title }` → creates card, returns new card
- [ ] `PATCH /api/cards/:id/move` → `{ toColumn, toIndex }` → moves card, runs policies and hooks
- [ ] `DELETE /api/cards/:id` → deletes card
- [ ] `POST /api/archive` → archives Done column
- [ ] `GET  /` → serves board HTML

### Frontend
- [ ] Board renders columns and cards identical to VSCode extension UI
- [ ] Drag-and-drop card movement works and calls `PATCH /api/cards/:id/move`
- [ ] Add card, edit card, delete card all work via API calls
- [ ] WIP limit violations shown to user (same message text as extension)
- [ ] Board auto-refreshes when files change on disk (polling or SSE)

### Server behavior
- [ ] Default port: 3737 (configurable via `--port` flag or `KANBAN_PORT` env var)
- [ ] `kanban serve` opens the board URL in the default browser automatically
- [ ] Server prints `Board running at http://localhost:3737` on startup
- [ ] Graceful shutdown on SIGINT

### Tests
- [ ] Route handlers tested with supertest (or equivalent) against a temp board directory
- [ ] `npm test --workspace=packages/kanban-web` passes

## Implementation Notes

- `packages/kanban-web/` created with Express server, 3 route groups (board, cards, archive)
- `public/adapter.js` shims `window.vscode.postMessage` into `fetch()` calls — board.js required only a 1-line change (remove `acquireVsCodeApi()`, use `window.vscode` instead)
- `public/index.html` mirrors the VSCode webview HTML structure; loads adapter.js before board.js
- All 3 built-in theme CSS files copied into `public/themes/`
- 3-second polling in adapter.js keeps board in sync with external file changes
- Policy violations on move return HTTP 409; adapter dispatches `policyViolation` message so board.js can surface it
- `kanban serve` command added to CLI — opens browser automatically, dynamic-imports `@personal-kanban/web`
- 15/15 web tests pass (supertest), 21/21 CLI tests pass
